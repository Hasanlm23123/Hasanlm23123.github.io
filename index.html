Problem solved: candidates and recruiters can quickly see how well a resume matches a role, which keywords are missing, and what concrete edits or interview prep steps make the biggest difference.

Stack: JavaScript, Groq API, Vercel serverless functions, streamed UI updates, and server-side PDF parsing for resume upload, job-link ingestion, and exported report generation.

Architecture Notes

  • User uploads a PDF or pastes resume content in the browser.
  • Vercel serverless routes parse input, build prompts, and call Groq server-side.
  • Structured output and streamed chunks return to the UI for scoring, rewrites, and comparisons.
  • The client renders recruiter snapshot, fit-score rationale, export, and saved-history flows.

Challenges + Learnings

  • Handled inconsistent PDF extraction and partial model JSON so analysis degrades gracefully.
  • Tightened prompt structure to reduce vague bullet rewrites and generic gap advice.
  • Balanced serverless constraints with streamed output so the app still feels immediate to use.
